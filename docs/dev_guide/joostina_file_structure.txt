Структура файловой системы Joostina CMS

Образоно файловую структуру Joostina можно разделить на 2 уровня:
1. Системные файлы ядра
2. Пользовательские данные и расширения для работы

Системные файлы - это файлы необходимые для нормлаьной работы ядра, подробнее:
/core
	- joostina.php - главный файл ядра
	- front.root.php - расширение ядра для работы в пространстве пользовательской части сайта - фронт
	- admin.root.php - расширение ядра для работы административной части - панель управления
	- autoloader.php - мастер автозагрузки классов и расширений системы
	- exception.php - мастер обработки системных ошибок и исключений
В каталоге core так же содержится подкаталог libraries, в котором находятся системные библиотеки расширяющие возможности ядра и обеспечивающие базовые возможности для решения типичных задач - работы с файлами, куками, базой данных или текстами.

Пользовательские данные - это файлы расширений, такие как компоненты и модули, и файлы медиа - загруженные файлы ( аттачи ) и файлы оформления шаблона - стили, скрипты и т.д.
Подробнее про каталог пользовательских данных:
app
	- bootstrap.php - файл настройки рабочего окружения ядра, системные константы, правила инициализации системы и т.д.
	- config.php - базовый файл конфигурации ядра и расширений
	- routes.php - правила роутинга и формирования ссылок. Всё управление ссылками системы и зависимости запускаемых расширений прописываются только в этом файле
	- autoload.php - правила загрущки классов системы. Что бы ядро лишний раз не искала расположение требуемого класса - его можно прописать в этом файле
	- events.php - навешивание обытий на системные функции
	- components - компоненты
	- modules - модули
	- templates - шаблоны
	- plugins - плагины
	- attachments - каталог загруженных пользовательских данных - аттачей

Каталог ядра, подкаталоги компонентов и шаблонов по необзодимости содержат сходный по структуре подкаталог media, который содержит следующие файлы:
media
	- images - картинки и изобраджения
	- js - javascript файлы
	- swf - флеш файлы
	- css - стилевые css файлы


В системе так же имеется каталог admin, который содержит стартовые файлы для доступа к панели управления. Сам http доступ к панели упралвегния по умолчанию выглядит как сайт.ру/admin

Базовая структура компонента выглядит следующим образом, на прмиере компонента упралвения пользователями users:
/app/components/users/
- controller.users.php - главный файл контроллер, обеспечивающий работу пользовательской части компонента
- controller.users.ajax.pjp - контроллер пользовательской части, отвечающий за выполнение действий через ajax
- controller.admin.users.php - контроллер, отвечающий за работу панели упралвения компонентом
- controller.admin.users.ajax.php - файл контроллера управляющий ajax функциями панели упралвения компонентом
- /media - структура каталога описана выше, медиа файлы необходимые для работы компонента пользователей
- /models - каталог с моделями компонента управления пользователями
- /models/model.users.php - модель работы с пользователями на фронте сайта
- /models/model.admin.users.php - модель работы с пользователями в панели управления
- /views - шаблоны оформления задач упралвения пользователями
- /views/index/default.php - шаблон соответсвет методу index в контроллере упралвения пользователями
